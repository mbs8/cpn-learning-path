# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Commands

```bash
npm run setup          # First-time setup: install deps, generate Prisma client, run migrations
npm run dev            # Start dev server with Turbopack at localhost:3000
npm run dev:daemon     # Start dev server in background, logs to logs.txt
npm run build          # Production build
npm run lint           # ESLint check
npm run test           # Run all tests with Vitest
npx vitest run src/path/to/__tests__/file.test.ts  # Run a single test file
npm run db:reset       # Reset database (destructive)
```

> Do NOT run `npm audit fix`. Dependencies are pinned to versions that work together; audit fix can break the app.

## Architecture

UIGen is a Next.js 15 app that lets users describe React components in chat and see a live preview generated by Claude.

### AI Pipeline

1. User sends a message → `ChatContext` (`src/lib/contexts/chat-context.tsx`) calls the chat API via Vercel AI SDK's `useChat`
2. `src/app/api/chat/route.ts` streams a response using `streamText` with two tools:
   - `str_replace_editor` (`src/lib/tools/str-replace.ts`): targeted line/block edits to files
   - `file_manager` (`src/lib/tools/file-manager.ts`): create/delete/rename files
3. Tool call results update the **VirtualFileSystem** via `FileSystemContext` (`src/lib/contexts/file-system-context.tsx`)
4. The updated virtual FS triggers re-render of the preview iframe

The provider (`src/lib/provider.ts`) falls back to a mock that returns canned components when `ANTHROPIC_API_KEY` is absent or set to the placeholder value.

### Virtual File System

All generated code lives in memory — nothing is written to disk. `VirtualFileSystem` (`src/lib/file-system.ts`) is a class that manages an in-memory file/directory tree. Projects are persisted to SQLite by serializing the VFS state alongside the chat message history.

### JSX Preview

`src/lib/transform/jsx-transformer.ts` compiles JSX using `@babel/standalone` at runtime and injects it into an iframe (`PreviewFrame`). It builds an import map so the generated components can reference React and Tailwind inside the sandboxed iframe.

### Auth

JWT sessions stored in httpOnly cookies (`src/lib/auth.ts` + `src/middleware.ts`). Anonymous users are supported — `src/lib/anon-work-tracker.ts` persists anonymous work to localStorage so it can be claimed after sign-up.

### Database

Prisma ORM with SQLite (`prisma/schema.prisma`). Schema has two models: `User` (email + bcrypt password) and `Project` (owner, chat messages JSON, serialized VFS JSON).

### UI Layout

Three-panel resizable layout in `src/app/main-content.tsx`:
- Left: `ChatInterface` (message list + input)
- Right top: `PreviewFrame` (live iframe) or `CodeEditor` (Monaco) toggled by tabs
- Right bottom: `FileTree` for navigating generated files

### State Management

No external state library. Two React contexts drive the app:
- `ChatContext` — chat messages, AI SDK integration, streaming state
- `FileSystemContext` — VFS contents, selected file, tool-call application logic

### Tech Stack

- Next.js 15 (App Router) · React 19 · TypeScript (strict)
- Tailwind CSS v4 (no `tailwind.config.*`; configured via `postcss.config.mjs`)
- shadcn/ui components (New York style) in `src/components/ui/`
- Vitest + Testing Library for unit/component tests
- Prisma + SQLite for persistence
- Vercel AI SDK (`ai` package) + `@ai-sdk/anthropic` for streaming
